## Overview

Optimize pattern priorities based on real-world usage metrics using the Learning Agent CLI commands.

The learning agent (Burler) monitors pattern effectiveness, generates improvement
proposals, and applies changes with safety mechanisms (circuit breakers,
autonomy levels, rollback support).


## Prerequisites

- Loom v1.2.0+
- Loom server running: `looms serve`
- Pattern library YAML files
- Sufficient usage data (50+ samples per pattern recommended for MEDIUM confidence)

> **Note:** All `looms learning` subcommands connect to the Loom gRPC server at `localhost:60051` by default. Use `--server` to override.

## Quick Start

Preview pattern tuning:

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --library=./pkg/patterns \
  --dry-run
```

Apply tuning:

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --library=./pkg/patterns
```

## Configuration

### Tuning Strategies

| Strategy | Max Adjustment | Use Case |
|----------|----------------|----------|
| `conservative` | +/-5 priority | Small tweaks |
| `moderate` | +/-15 priority | General use |
| `aggressive` | +/-30 priority | Development, major changes |

### Legacy Optimization Weights

These flags (`--quality-weight`, `--cost-weight`, `--latency-weight`) are used for patterns that do not have judge scores. For patterns with judge scores, use dimension weights instead (see below).

| Weight | Default | Description |
|--------|---------|-------------|
| `quality-weight` | 0.5 | Success rate importance |
| `cost-weight` | 0.3 | LLM cost importance |
| `latency-weight` | 0.2 | Response time importance |

### Judge Dimension Weights

For patterns with judge scores, use `--dimension-*` flags for multi-dimensional tuning. These override legacy weights when judge scores are available.

| Flag | Description |
|------|-------------|
| `--dimension-quality` | Quality score weight (0.0-1.0) |
| `--dimension-safety` | Safety score weight (0.0-1.0) |
| `--dimension-cost` | Cost score weight (0.0-1.0) |
| `--dimension-domain` | Domain compliance weight (0.0-1.0) |
| `--dimension-performance` | Performance score weight (0.0-1.0) |
| `--dimension-usability` | Usability score weight (0.0-1.0) |

Dimension weights are normalized if they do not sum to 1.0.

### Confidence Levels

| Confidence | Sample Size | Adjustment |
|------------|-------------|------------|
| LOW | <50 samples | 50% of strategy range |
| MEDIUM | 50-200 samples | 100% of strategy range |
| HIGH | >200 samples | 100% of strategy range |

## Common Tasks

### Analyze Pattern Effectiveness

View pattern performance metrics:

```bash
looms learning analyze --domain=sql --window=48
```

The default time window is 24 hours. Use `--window` to specify a different window in hours.

Expected output:
```
Pattern Analysis (12 patterns)
------------------------------
sequential_scan_detection
  Usage: 245 (233 success, 12 failures)
  Success Rate: 95.1%
  Avg Cost: $0.0023 | Avg Latency: 450ms
  Recommendation: PROMOTE

query_rewrite
  Usage: 67 (52 success, 15 failures)
  Success Rate: 77.6%
  Avg Cost: $0.0045 | Avg Latency: 890ms
  Recommendation: DEMOTE
```

Recommendations are one of: PROMOTE, KEEP, DEMOTE, REMOVE, INVESTIGATE.

### Tune Pattern Priorities

The `--library` flag is required and must point to your pattern library YAML files.

Preview changes first:

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --library=./pkg/patterns \
  --dry-run
```

Apply changes:

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --library=./pkg/patterns
```

### List Improvement Proposals

View pending improvement proposals generated by the learning agent:

```bash
looms learning proposals
looms learning proposals --status=pending --domain=sql
looms learning proposals --status=applied --limit=10
```

Status filter options: `pending`, `applied`, `rolled_back`, `rejected`.

### Apply an Improvement

Apply a specific improvement proposal by its ID:

```bash
looms learning apply abc123-def456-ghi789
```

The improvement is applied and pattern hot-reload is triggered to activate the new pattern version. Respects the configured autonomy level (manual, human approval, full auto).

### Rollback an Improvement

Rollback a previously applied improvement:

```bash
looms learning rollback abc123-def456-ghi789
```

Restores the previous pattern version and triggers hot-reload.

### View Improvement History

Get the audit trail of all learning agent decisions:

```bash
looms learning history
looms learning history --domain=sql --status=applied
looms learning history --agent=sql-optimizer-abc123 --limit=50
```

### Stream Metrics

Monitor real-time pattern metrics:

```bash
looms learning stream --domain=sql
```

Press **Ctrl+C** to stop.

## Examples

### Example 1: Dry Run Tuning

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --library=./pkg/patterns \
  --dry-run

# Output:
# Pattern Tuning Configuration
#   Domain: sql
#   Strategy: moderate
#   Mode: DRY RUN
#
# sequential_scan_detection (sql)
#   Confidence: HIGH
#   priority: 60 -> 75
#   Rationale: High success rate 95%
#
# Tuning Summary
#   Patterns Analyzed: 12
#   Patterns Tuned: 8
#   Promoted: 5
#   Demoted: 2
#   Unchanged: 5
```

### Example 2: Quality-Focused Optimization

Prioritize accuracy over cost using legacy weights:

```bash
looms learning tune \
  --domain=sql \
  --strategy=aggressive \
  --quality-weight=0.8 \
  --cost-weight=0.1 \
  --latency-weight=0.1 \
  --library=./pkg/patterns
```

### Example 3: Cost-Focused Optimization

Minimize LLM costs using legacy weights:

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --quality-weight=0.3 \
  --cost-weight=0.6 \
  --latency-weight=0.1 \
  --library=./pkg/patterns
```

### Example 4: Dimension-Weighted Tuning

Use judge dimension weights for multi-dimensional tuning (preferred over legacy weights when judge scores are available):

```bash
looms learning tune \
  --domain=sql \
  --strategy=moderate \
  --dimension-quality=0.6 \
  --dimension-safety=0.3 \
  --dimension-cost=0.1 \
  --library=./pkg/patterns \
  --dry-run
```

Patterns with judge scores use the dimension-weighted scoring. Patterns without judge scores fall back to legacy weights automatically.

### Example 5: Production Workflow

```bash
# 1. Analyze current performance
looms learning analyze --domain=sql --window=168  # 1 week

# 2. Review proposals
looms learning proposals --domain=sql

# 3. Preview conservative tuning
looms learning tune \
  --domain=sql \
  --strategy=conservative \
  --library=/opt/loom/patterns \
  --dry-run

# 4. Apply if improvements are meaningful
looms learning tune \
  --domain=sql \
  --strategy=conservative \
  --library=/opt/loom/patterns

# 5. Monitor results
looms learning stream --domain=sql

# 6. Rollback if needed
looms learning history --domain=sql --status=applied
looms learning rollback <improvement-id>
```

## Troubleshooting

### No Patterns Found

Verify metrics collection is working:

```bash
looms learning analyze --domain=sql
```

Check library path:

```bash
ls ./pkg/patterns/**/*.yaml
```

### All Patterns Have LOW Confidence

Collect more usage data. Target 50+ usages per pattern for MEDIUM confidence.

```bash
# Use conservative strategy with low confidence
looms learning tune --strategy=conservative --library=./pkg/patterns
```

### Weights Sum Warning

The CLI auto-normalizes weights. Example:

```bash
# Input: 0.6 + 0.5 + 0.3 = 1.4
# Normalized: 0.43 + 0.36 + 0.21 = 1.0
```

This applies to both legacy weights and dimension weights.

### Pattern Name Mismatch

Check pattern names in metrics match YAML files:

```bash
looms learning analyze --domain=sql | grep pattern_name
grep -r "name: pattern-name" ./pkg/patterns/
```

### Connection Errors

If you get a connection error, verify the Loom server is running:

```bash
looms serve
```

The default server address is `localhost:60051`. Override with `--server`:

```bash
looms learning analyze --server=myhost:60051 --domain=sql
```
