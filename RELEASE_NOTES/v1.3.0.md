## Summary

v1.3.0 is a feature release focused on **memory, tasks, and skills**. It adds salience-driven graph memory, an LLM-assisted task-decomposition and kanban system, a substantially expanded skills system (declarative bindings, a hierarchical router, task emission, and import of Anthropic-style skill directories), and a 3-tier self-healing error-recovery path in the agent loop. It also ships a load-testing framework with measured throughput improvements, the LongMemEval benchmark harness with an LLM-judge criteria fix, a span-exporter interface, agent presets / workflow templates, a local-Ollama profile, Bedrock bearer-token auth, and the Teradata pattern library expanded from 34 to 84 patterns.

This release contains **breaking changes** — see the next section before upgrading.

- **54 commits** since v1.2.0 (13 features, 12 fixes, dependency/CI maintenance)
- **3739 test functions** across 393 test files (up from 3112/319), 0 race conditions
- **9 new database migrations** (SQLite + PostgreSQL); applied automatically on first start

## Breaking Changes

| Change | Impact | Action |
|--------|--------|--------|
| **IntentCategory enum removed** (#137) | Patterns no longer use the enum | Declare freeform `intents` strings in pattern YAML (YAML fallback preserved) |
| **Ollama default timeout** (#188) | Zero/unset is now provider-driven, not a 60s fallback | If you relied on the implicit 60s, set an explicit timeout |
| **LLM catalog model IDs** (#156) | Legacy short-prefix map removed for `claude-opus-4` / `claude-sonnet-4` / `claude-haiku-4` | Use catalog-exact model IDs |
| **Graph memory extraction default-on** (#137) | Extraction subprocess runs when graph memory is enabled, raising LLM-call volume | Opt out with `enable_extraction: false` |
| **Skills config** | `enabled_skills` / `disabled_skills` deprecated; skills emit tasks by default | Move to `bindings[]` (compat shim retained); opt out of task emission with `emit_tasks: false` (skill) or `tasks_enabled: false` (agent) |
| **Auto-applied DB migrations** | SQLite `000005` (embedding) and `000006` (event_date) apply on first start | Back up databases before upgrading |

### Migration Notes

- **Event-date anchoring over gRPC is not yet available.** `event_date` / `event_date_confidence` are not fields on the proto `GraphMemory` message and are not round-tripped by `convert.go`. Event dates work via the internal Go layer only. Do not expect gRPC clients to read them in this release.
- **Ollama timeout:** if you depended on the implicit 60s timeout, set the provider/tool timeout explicitly — zero/unset now means provider-driven.
- **Model IDs:** if you reference `claude-opus-4` / `claude-sonnet-4` / `claude-haiku-4` by legacy short prefixes, update configs to catalog-exact IDs after the #156 catalog rewrite.
- **Bedrock SSO:** with IAM Identity Center / SSO you can now supply a bearer token via CLI/env/keyring (`bedrock_bearer_token`) to bypass SigV4. It is supplied at runtime only and is not persisted to config files.
- **Skills:** replace `enabled_skills` / `disabled_skills` with `bindings[]`. To stop skills auto-emitting tasks, set `emit_tasks: false` (per skill) or `tasks_enabled: false` (agent level).
- **Databases auto-migrate** (`000005` / `000006`) on first start of the upgraded server; back up first per the troubleshooting guide.

## Added Features

### Salience-Driven Graph Memory (#133, #137, #159)
`GraphMemoryService` with 15 RPCs (entity/edge CRUD, remember/recall/forget/supersede/consolidate, `ContextFor`, `GetGraphStats`), salience scoring with exponential time decay, an asynchronous LLM extraction subprocess, and a `graph_memory` tool auto-injected when a store is configured (default-on, opt-out). Event-date anchoring emits absolute ISO dates with confidence levels and prepends `[YYYY-MM-DD]` to context. ⚠️ Event dates are available via the internal Go layer only, not over gRPC (see Migration Notes). SQLite and PostgreSQL backends with FTS5, soft deletes, and user-scoped RLS.

### LLM-Assisted Task Decomposition & Agent Kanban (#141)
`TaskService` proto (Task/TaskBoard/TaskLane, 5 enums, 19 RPCs defined), a `task_board` agent tool (10 actions), a task Manager with DFS cycle detection / auto status propagation / WIP limits / compaction, and an LLM-assisted Decomposer (backward/forward/parallel strategies, retry-on-invalid-JSON, rollback). Dependency-aware SQLite + PostgreSQL storage. `OutputRetryPolicy` added to `collaboration.proto`. ⚠️ 11 of 19 RPCs are served (see Known Limitations).

### Skills System Expansion (#174, #182, #183, #184, #195)
Declarative skill bindings (exact/glob/label matching), a hierarchical PageIndex-style router with session caching, skill task emission with author-defined decompositions, end-of-turn task-board hygiene, `SkillsImportService` gRPC with post-write router reload, import of Anthropic-style Agent Skill directories, and per-skill risk levels. ⚠️ Risk levels are recorded and block when `require_approval=true`; an interactive approval flow is not yet implemented.

### Self-Healing Error Recovery (#177)
A 3-tier recovery path in the conversation loop (output-token circuit breaker → context trim; tool circuit breaker → local tool disable; token-budget pressure → aggressive trim), default-on with opt-out via `behavior.enable_self_healing: false`. One-shot per error type per conversation; surfaces a `RecoverableError` to upper layers when exhausted.

### Load Testing Infrastructure & Throughput Optimization (#139)
A zero-token mock LLM provider + gRPC harness with 15 load/scalability scenarios. Optimizations: O(1) per-layer token-count caching, a channel-based encoder pool, and a read-lock session fast path. Measured on Apple M4 Pro (14 cores): **2,200 req/s peak at 160 workers**; **11,023 req/s on fresh sessions vs 187 req/s on a shared session** (59x contention gap, by design — a single conversation serializes); throughput flat at ~2,400 req/s across 100–10,000 sessions. Full methodology in `docs/testing/load-test-report.md`.

### LongMemEval Benchmark & Judge Criteria Fix (#159)
`loom-longmemeval` CLI harness (ingest / multi-session / context-stuffing modes). The LLM judge now honors `JudgeConfig.Criteria` by routing to a generic criteria-driven prompt instead of a hardcoded SQL-evaluation template. Vector embedding infrastructure (`Embedder` interface + OpenAI implementation; SQLite brute-force `VectorRecall`).

### Span Exporter & Workflow Session Continuity (#132)
A `SpanExporter` interface (`ExportSpans`/`ForceFlush`/`Shutdown`) for external trace export, plus scheduled-workflow session continuity via `ScheduledSessionMode` (`NEW`/`RESUME`) using a stable `workflow_id`.

### Agent Presets & Workflow Templates
8 agent presets and 7 workflow templates with defaults registries, served via `ListAgentPresets`, `ListWorkflowTemplates`, and `CreateWorkflowFromTemplate` (idempotent by agent name).

### Local-Ollama Profile & Tool-Surface Controls (#193)
Per-role LLMs wired in the static-YAML loader, `llm.concurrency_limit` (set to 1 for single-slot local Ollama), `tools.none` to restrict an agent to its declared tools, and a local-Ollama config example.

### Teradata Patterns (#142) & Bedrock Bearer-Token Auth
Teradata patterns expanded 34 → 84 with 4 new skills. Optional Bedrock bearer-token auth (IAM Identity Center / SSO) that bypasses SigV4, supplied via CLI/env/keyring (runtime only).

## Known Limitations

- **Graph memory event dates over gRPC** — not transmittable; `event_date` fields are absent from the proto `GraphMemory` message and from `convert.go` round-trip. Works via the internal Go layer only.
- **Graph memory conversation-turn cadence** — `conversation_extraction_cadence` parses from config but is currently inert; only user-input and tool-execution triggers fire.
- **TaskService gRPC surface** — 11 of 19 RPCs served. Not yet implemented: `UpdateTask`, `DeleteTask`, `ReleaseTask`, `TransitionTask`, `AddDependency`, `RemoveDependency`, `GetBlockedTasks`, `DecomposeTask`, `GetTaskContext`. Decomposition is available via the `task_board` tool action.
- **Vector recall** — SQLite `VectorRecall` is brute-force O(n) (~100K memories/agent); PostgreSQL pgvector path is stubbed (not-implemented).
- **Skill risk levels** — used for block-when-`require_approval`; interactive HITL approval flow not yet implemented.

### Step 2: Push the release

> The push is the point of no return — it triggers the public GitHub Release workflow and package-manager publishing.

```bash
git push origin main
git push origin v1.3.0
```

This triggers:
- `.github/workflows/release.yml` — builds 12 binaries (loom / looms / loom-standalone × linux-amd64 / darwin-amd64 / darwin-arm64 / windows-amd64), GPG-signs checksums, builds the combined Windows package, and generates SLSA provenance attestations

The workflow's first job re-verifies that the tag matches the `VERSION` file and that all version files are consistent; a mismatch fails the release.

## Verification

After the release completes:

```bash
# 1. Download a binary + checksum + signature
wget https://github.com/teradata-labs/loom/releases/download/v1.3.0/loom-linux-amd64.tar.gz
wget https://github.com/teradata-labs/loom/releases/download/v1.3.0/loom-linux-amd64.tar.gz.sha256
wget https://github.com/teradata-labs/loom/releases/download/v1.3.0/loom-linux-amd64.tar.gz.sha256.asc

# 2. Import the public key and verify the signature
curl -sL https://raw.githubusercontent.com/teradata-labs/loom/main/loom-release-key.asc | gpg --import
gpg --verify loom-linux-amd64.tar.gz.sha256.asc   # Good signature from "Loom Release Bot"

# 3. Verify the checksum
sha256sum -c loom-linux-amd64.tar.gz.sha256       # loom-linux-amd64.tar.gz: OK

# 4. Verify SLSA provenance
gh attestation verify loom-linux-amd64.tar.gz --owner teradata-labs

# 5. Verify the binary reports the right version
tar xzf loom-linux-amd64.tar.gz && ./loom-linux-amd64 --version   # v1.3.0
```

## Rollback

If something is wrong after tagging but before pushing:

```bash
git tag -d v1.3.0                      # delete the local tag
git reset --hard HEAD~2                # drop the bump + fix commits (verify count first)
```

If the tag was already pushed, do **not** delete or move it — published package versions cannot be overwritten. Cut a new patch release (v1.3.1) with the fix instead.
